mxnet安装

mxnet安装很简单就是下好所有依赖，将dll位置加入path，使用cmake配置所有依赖项位置，另外build目标要选择vs2013 64bit
编译完成后或下载预编译版本，cd python python setup install即可，

善用C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\bin\dumpbin.exe来解决dll依赖

mxnet需要在path里指定依赖的dll位置

anaconda命令行居然会因为path里路径过多而出错，出错后在命令行里的PATH为空

C:\ProgramData\Oracle\Java\javapath\java.exe会覆盖JAVAHOME所指定的Java版本，没屁用可删

indri坑总结

  1. 在Field中dog.title,header被描述为出现在title或header中的dog，但实际上根据reference中的描述title和dog并非或的关系而是与关系
  2. trectext不一定要按照doc docno的格式可以自定义格式 参考 好像不会自动添加
  3. 在indri5.10时曾经遇到baseline选项不能为okapi的情况，5.11好像修复了 P.S indri介绍baseline选项的文件不在wiki里在安装文件夹的doc文件中的index里
  4. beta与dirichletbeta与dirichlet2beta与dirichlet3后验=先验*似然 若先验后验形式一致则称共轭，但是似然不是教课书上的分布，因为其数据已知而参数未知，与教科书相反。
  5. beta与dirichlet期望计算
  6. indri中文伪反馈bug大意就是不注释源代码中的isword函数的话，所有中文字符都会被过滤，不加入伪反馈
  7. indrirunquery -printQuery=true可检查伪反馈查询

linux注意事项

  1. tar -xf自动识别格式没必要写全
  2. yum install 先update
  3. find 查找 which查找命令对于文件（windows下叫where）
  4. sudo nohup nice -n -20 cmd & nohup不允许terminal发送hangup给命令进程（防止关闭terminal导致进程结束）nice)指定命令进程优先级-20最高 19最低 命令后加&可以后台运行命令，使得我们可以继续在terminal输入，当然这样就看不到命令输出了，如果要看到输出可以重定向输出到文件然后使用tail来跟踪(tail -n 100 -f file)
